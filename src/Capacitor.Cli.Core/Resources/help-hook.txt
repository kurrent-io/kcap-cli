kcap hook — dispatch a vendor hook event

Usage: kcap hook --<vendor>

Description:
  Invoked by a coding agent's hook system, NOT by users directly. Reads
  the hook payload as JSON from stdin (including `hook_event_name`) and
  forwards it to the kcap server. The vendor flag selects the per-agent
  dispatcher; the event is taken from the payload.

  The command is registered via 'kcap setup' or
  'kcap plugin install [--claude|--codex|--cursor|--copilot|--kiro]', which
  writes the appropriate hook configuration for the detected vendor.

Options:
  --claude    Dispatch the event as a Claude Code hook.
  --codex     Dispatch the event as a Codex CLI hook.
  --cursor    Dispatch the event as a Cursor IDE hook.
  --copilot   Dispatch the event as a GitHub Copilot CLI hook. Copilot
              payloads carry no event-name field, so the installer embeds
              it in the registered command (--event <name>).
  --kiro      Dispatch the event as an AWS Kiro CLI hook. kcap subscribes
              only to agentSpawn (session start; fires every prompt, deduped
              server-side); session-end is synthesized by the watcher on
              kiro-cli exit. Always exits 0 with empty stdout.

Exit codes:
  0   Normal exit (including budget expiry, malformed payload, and
      disabled-session suppression).
  1   No vendor flag provided, missing hook_event_name, or server error.
