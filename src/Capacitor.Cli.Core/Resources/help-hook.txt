kcap hook — dispatch a vendor hook event

Usage: kcap hook --<vendor>

Description:
  Invoked by a coding agent's hook system, NOT by users directly. For most
  vendors it reads the hook payload as JSON from stdin (including
  `hook_event_name`) and forwards it to the kcap server; the vendor flag
  selects the per-agent dispatcher and the event is taken from the payload.

  Pi and OpenCode are the exceptions: they have no shell hooks, so their
  dispatch is argument-driven (no stdin payload) — see --pi / --opencode below.

  The command is registered via 'kcap setup' or
  'kcap plugin install [--claude|--codex|--cursor|--copilot|--gemini|--kiro|--pi|--opencode|--antigravity]',
  which writes the appropriate hook configuration (or, for Pi/OpenCode/Antigravity,
  the plugin/hooks-block) for the vendor.

Options:
  --claude    Dispatch the event as a Claude Code hook.
  --codex     Dispatch the event as a Codex CLI hook.
  --cursor    Dispatch the event as a Cursor IDE hook.
  --copilot   Dispatch the event as a GitHub Copilot CLI hook. Copilot
              payloads carry no event-name field, so the installer embeds
              it in the registered command (--event <name>).
  --gemini    Dispatch the event as a Gemini CLI hook. Self-routes on the
              payload's hook_event_name (SessionStart/SessionEnd/Notification).
  --kiro      Dispatch the event as an AWS Kiro CLI hook. kcap subscribes
              only to agentSpawn (session start; fires every prompt, deduped
              server-side); session-end is synthesized by the watcher on
              kiro-cli exit. Always exits 0 with empty stdout.
  --pi        Dispatch a Pi (badlogic/pi-mono) lifecycle event. Pi has no
              shell hooks; the shipped kcap.ts extension invokes this with
              arguments instead of a stdin payload:
                kcap hook --pi --event <session-start|session-end> \
                  --file <session-file> [--cwd <cwd>] [--reason <reason>]
              The session id, cwd, and start time are read from the Pi
              session JSONL header.
  --opencode  Dispatch an SST OpenCode lifecycle event. OpenCode has no shell
              hooks; the shipped kcap.ts plugin invokes this with arguments
              instead of a stdin payload:
                kcap hook --opencode --event session-start \
                  --session <id> --file <transcript-file> [--cwd <cwd>]
              The plugin writes {info,parts} JSONL to the transcript file; the
              watcher tails it (vendor=opencode) and synthesizes session-end on
              opencode exit.
  --antigravity  Dispatch a Google Antigravity control-hook event. Antigravity
              payloads carry no event-name field, so the event is passed as a
              positional arg with the payload on stdin:
                kcap hook --antigravity <PreInvocation|PostInvocation|Stop|
                  PreToolUse|PostToolUse>
              kcap acts on the first PreInvocation of a conversation (spawns a
              watcher tailing its transcript_full.jsonl, vendor=antigravity) and
              synthesizes session-end on idle. Other events are no-ops.

Exit codes:
  0   Normal exit (including budget expiry, malformed payload, and
      disabled-session suppression).
  1   No vendor flag provided, missing hook_event_name, or server error.
