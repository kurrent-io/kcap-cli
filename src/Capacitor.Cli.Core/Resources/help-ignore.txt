kcap ignore — Exclude paths from session recording

Usage: kcap ignore <path>
       kcap ignore --list
       kcap ignore --remove <path>

Marks <path> as excluded for the active profile. Any coding-agent session whose
working directory is the path itself, or any directory inside it, is silently
dropped — no hook payload is sent and no transcript is imported via
`kcap import`.

Path resolution:
  - `.` and `~` are expanded to absolute paths.
  - Trailing separators are stripped.
  - Symlinks are resolved so a worktree symlink and its target match.

Scope:
  Entries are stored on the **active profile** (see `kcap profile show`).
  Switching profiles with `kcap use` switches the ignore list too. To
  share the same list across profiles, run `kcap ignore` while each
  profile is active.

Examples:
  kcap ignore .                            # ignore the current directory
  kcap ignore ~/code/secret-project        # ignore a specific tree
  kcap ignore --list                       # show all ignored paths
  kcap ignore --remove ~/code/secret-project
