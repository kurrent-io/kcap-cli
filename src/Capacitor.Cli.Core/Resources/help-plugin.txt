kcap plugin — Manage hooks and skills for Claude Code, Codex CLI, Cursor, Copilot CLI, Gemini CLI, Kiro CLI, Pi, OpenCode, Antigravity, and other agents

Usage: kcap plugin <subcommand> [options]

Subcommands:
  install             Register the plugin / hooks / skills
  remove              Remove the plugin / hooks / skills

Options:
  --project           Apply hooks to current project only (default: user-wide).
                      Skills are always user-wide; --project only affects hooks.
  --codex             Target Codex CLI: install hooks AND agent skills.
  --cursor            Target Cursor IDE: install hooks to ~/.cursor/hooks.json.
                      Detected by user-dir presence, not PATH, so users without
                      the cursor shell command are covered.
  --copilot           Target GitHub Copilot CLI: write kcap's own hooks file to
                      ~/.copilot/hooks/kcap.json (honours $COPILOT_HOME). Copilot
                      merges every *.json in that directory, so user-authored
                      hook files are never touched.
  --gemini            Target Gemini CLI: merge kcap's hooks into the SHARED
                      ~/.gemini/settings.json (honours $GEMINI_CLI_HOME). User-authored
                      hook entries and every other settings key are preserved; if
                      the file exists but isn't valid JSON the install fails closed
                      and leaves it untouched.
  --kiro              Target AWS Kiro CLI: clone your current default Kiro agent
                      (preserving its tools) to ~/.kiro/agents/kcap.json, add
                      kcap's agentSpawn hook, and make it your default so every
                      session is captured (honours $KIRO_HOME). Needs kiro-cli on
                      PATH. Session-end is synthesized on exit.
  --pi                Target Pi (badlogic/pi-mono): install the live-ingest
                      extension to ~/.pi/agent/extensions/kcap.ts. Pi has no
                      shell hooks; the extension auto-loads on `pi` startup and
                      streams each session. User-wide only — --project has no
                      effect with --pi.
  --opencode          Target SST OpenCode: install the live-ingest plugin to
                      ~/.config/opencode/plugins/kcap.ts. OpenCode has no shell
                      hooks; the plugin auto-loads on `opencode` startup and
                      streams each session. User-wide only — --project has no
                      effect with --opencode.
  --antigravity       Target Google Antigravity: install the kcap capture plugin
                      to ~/.gemini/config/plugins/kcap/ (a plugin.json manifest +
                      hooks.json). Antigravity has no shell hooks; the GUI loads
                      plugins at startup and streams each conversation. User-wide
                      only — --project has no effect with --antigravity.
  --skills            Install only the agent-agnostic skills to ~/.agents/skills/.
                      Use this if you only have Cursor (or another agent that
                      reads ~/.agents/skills/) and don't want Codex hooks.
  --skip-codex-network-access
                      With --codex: skip enabling Codex sandbox network access.
                      By default `install --codex` also allows Codex's
                      workspace-write sandbox to reach your configured Capacitor
                      server(s) (a network_proxy allowlist in
                      ~/.codex/config.toml) so kcap skills work. The
                      --if-installed refresh path never touches this, so the npm
                      postinstall hook leaves config.toml alone.
  --skip-cursor-mcp   With --cursor: skip registering the kcap MCP servers in
                      ~/.cursor/mcp.json. By default `install --cursor` also
                      registers all 4 kcap MCP servers there (non-destructive,
                      idempotent — preserves any user-authored servers) so
                      Cursor picks them up with no manual JSON edit.
  --skip-copilot-mcp  With --copilot: skip registering the kcap MCP servers in
                      ~/.copilot/mcp-config.json. By default `install --copilot`
                      also registers all 4 kcap MCP servers there (non-destructive,
                      idempotent — preserves any user-authored servers) so Copilot
                      picks them up with no manual JSON edit.
  --skip-copilot-instructions
                      With --copilot: skip installing kcap's steering-instructions
                      block into ~/.copilot/copilot-instructions.md. By default
                      `install --copilot` writes a small kcap-owned, marker-delimited
                      block (non-destructive — preserves your own instructions) that
                      nudges Copilot to use the kcap tools for why/history questions.
  --skip-gemini-mcp   With --gemini: skip registering the kcap MCP servers in the
                      shared ~/.gemini/settings.json. By default `install --gemini`
                      also registers all 4 kcap MCP servers there (non-destructive,
                      idempotent — preserves your other Gemini settings + servers)
                      so Gemini picks them up with no manual JSON edit.
  --skip-gemini-instructions
                      With --gemini: skip installing kcap's steering-instructions
                      block into Gemini's global ~/.gemini/GEMINI.md context file.
                      By default `install --gemini` writes a small kcap-owned,
                      marker-delimited block (non-destructive — preserves your own
                      instructions) that nudges Gemini to use the kcap tools for
                      why/history questions.
  --skip-antigravity-mcp
                      With --antigravity: skip registering the kcap MCP servers in
                      ~/.gemini/config/mcp_config.json (Antigravity's own MCP file).
  --skip-antigravity-instructions
                      With --antigravity: skip installing kcap's steering block into
                      the shared ~/.gemini/GEMINI.md.
  --skip-antigravity-skills
                      With --antigravity: skip copying the kcap skills into
                      ~/.gemini/skills (where Antigravity reads them, not ~/.agents/skills).
  --if-installed      Refresh-only mode: no-op unless the user has previously
                      installed the target (marker file or pre-marker entry
                      detected). Used by the npm postinstall hook to refresh
                      command strings + plugin paths on upgrade without forcing
                      installs onto fresh systems. Errors are swallowed so npm
                      install never fails. Harmless to call by hand.

Notes:
  Skills install to ~/.agents/skills/kcap-{recap,errors,hide,disable,validate-plan}/.
  Codex, Cursor, and any other agent that honors the .agents/skills convention
  will pick them up automatically.

  --codex also writes hooks to ~/.codex/hooks.json (or <repo>/.codex/hooks.json
  with --project). MCP server registration is handled separately by the Codex
  plugin manifest and is not affected by this command.

  --cursor writes all 8 supported hooks to ~/.cursor/hooks.json (merged with
  any existing entries) and then registers all 4 kcap MCP servers in
  ~/.cursor/mcp.json (non-destructive, idempotent — see --skip-cursor-mcp to
  opt out). Cursor uses a single user-scope hooks.json/mcp.json — there is no
  project-scope variant, so --project has no effect with --cursor.

  install --codex and install --skills both clean up legacy ~/.codex/skills/kcap-*
  folders from prior installer versions.

  --copilot writes the four events kcap subscribes to (sessionStart, sessionEnd,
  agentStop, notification) with the event name embedded in each command —
  Copilot hook payloads don't carry one — registers all 4 kcap MCP servers in
  ~/.copilot/mcp-config.json (see --skip-copilot-mcp to opt out), and installs a
  kcap-owned steering block into ~/.copilot/copilot-instructions.md (see
  --skip-copilot-instructions to opt out). All three are non-destructive and
  idempotent. Copilot loads hook + MCP + instructions config at startup, so
  restart any running copilot session after install. Copilot has no project-scope
  hooks dir, so --project has no effect with --copilot.

  --gemini merges SessionStart/SessionEnd/Notification entries into the shared
  ~/.gemini/settings.json (like --cursor's shared hooks.json — your other Gemini
  settings and hooks are kept), registers all 4 kcap MCP servers in that same
  settings.json (see --skip-gemini-mcp to opt out), and installs a kcap-owned
  steering block into Gemini's global ~/.gemini/GEMINI.md context file (see
  --skip-gemini-instructions to opt out). All three are non-destructive and
  idempotent. Gemini loads hook + MCP config and the GEMINI.md context at
  startup, so restart any running gemini session after install. Gemini has no
  project-scope settings dir, so --project has no effect with --gemini.

  --kiro clones your current default Kiro agent into ~/.kiro/agents/kcap.json
  (preserving its tools — a minimal agent would lose tool access), injects the
  agentSpawn hook, and sets it as your default agent (chat.defaultAgent in
  ~/.kiro/settings/cli.json) because Kiro hooks fire only for the ACTIVE agent —
  there is no global hook. Needs kiro-cli on PATH to clone. Restart kiro-cli to
  pick it up. remove restores your previous default and deletes kcap.json. Kiro
  has no project-scope hooks, so --project has no effect with --kiro.

  --pi writes a single extension file (~/.pi/agent/extensions/kcap.ts) instead
  of a hooks file — Pi exposes no shell hooks, only an in-process extension API.
  The extension shells out to `kcap hook --pi` on session start/end, so kcap
  must be on PATH. Pi loads extensions at startup, so restart any running `pi`
  session after install. Pi has no project-scope extensions dir, so --project
  has no effect with --pi.

  --antigravity installs the kcap plugin to ~/.gemini/config/plugins/kcap/ — a
  plugin.json manifest (which the GUI requires to load the dir) plus a hooks.json
  kcap block (preserving any hook blocks you authored). It also registers the 4 kcap
  MCP servers in ~/.gemini/config/mcp_config.json (Antigravity's OWN MCP file, not the
  Gemini CLI's settings.json — see --skip-antigravity-mcp), installs a steering block
  into the shared ~/.gemini/GEMINI.md (see --skip-antigravity-instructions), and copies
  the kcap skills into ~/.gemini/skills (Antigravity does NOT read ~/.agents/skills — see
  --skip-antigravity-skills). Antigravity exposes no shell hooks, only control hooks. Each
  event runs `kcap hook --antigravity <Event>`, so kcap must be on PATH. The GUI loads
  plugins at startup, so restart it after install. User-wide only — --project has no effect
  with --antigravity.

  --opencode writes a single plugin file (~/.config/opencode/plugins/kcap.ts)
  instead of a hooks file — OpenCode exposes no shell hooks, only an in-process
  plugin API. The plugin shells out to `kcap hook --opencode` on session start
  and writes the {info,parts} transcript the watcher tails, so kcap must be on
  PATH. OpenCode loads plugins at startup, so restart any running `opencode`
  session after install. OpenCode has no project-scope plugins dir, so --project
  has no effect with --opencode.

  --if-installed is scoped to user-wide installs only — the npm postinstall hook
  does not refresh --project installs.

Examples:
  kcap plugin install                       # Install Claude Code plugin (user)
  kcap plugin install --skills              # Install agent skills only (~/.agents/skills/)
  kcap plugin install --codex               # Install Codex hooks + agent skills (user)
  kcap plugin install --project --codex     # Codex hooks in <repo>/.codex/hooks.json, skills user-wide
  kcap plugin install --cursor              # Install Cursor hooks (~/.cursor/hooks.json)
  kcap plugin install --copilot             # Install Copilot hooks (~/.copilot/hooks/kcap.json)
  kcap plugin install --gemini              # Install Gemini hooks (merge into ~/.gemini/settings.json)
  kcap plugin install --kiro                # Clone default Kiro agent + hooks, set as default
  kcap plugin install --pi                  # Install Pi extension (~/.pi/agent/extensions/kcap.ts)
  kcap plugin install --opencode            # Install OpenCode plugin (~/.config/opencode/plugins/kcap.ts)
  kcap plugin install --antigravity         # Install Antigravity capture plugin (~/.gemini/config/plugins/kcap/)
  kcap plugin install --skills --if-installed   # Refresh skills if previously installed (used by npm postinstall)
  kcap plugin install --codex --if-installed    # Refresh Codex hooks if previously installed (used by npm postinstall)
  kcap plugin install --if-installed            # Refresh Claude plugin registration if previously installed
  kcap plugin remove --codex                # Remove Codex hooks + agent skills + legacy ~/.codex/skills
  kcap plugin remove --cursor               # Remove Cursor hooks from ~/.cursor/hooks.json
  kcap plugin remove --copilot              # Delete ~/.copilot/hooks/kcap.json
  kcap plugin remove --gemini               # Remove kcap hooks from ~/.gemini/settings.json
  kcap plugin remove --kiro                 # Restore previous default agent, delete kcap.json
  kcap plugin remove --opencode             # Delete ~/.config/opencode/plugins/kcap.ts
  kcap plugin remove --antigravity          # Remove the kcap plugin from ~/.gemini/config/plugins/kcap/
  kcap plugin remove --skills               # Remove agent skills + legacy ~/.codex/skills
